# 🌐 Novae Rog: Front-End Architecture & Discovery Interface Specification

This document details the front-end strategies and interface designs needed for both **humans** (developers and liquidity providers) and **autonomous AI agents** to easily find, trust, and implement **Novae Rog**.

*Note: This is a standalone design concept and is kept separate from the core `system-spec.md` to keep the initial development focus strictly on the fundamental smart contracts and backend agents.*

---

## 🧑‍💻 1. The Human Interface Layer (DApp Dashboard)

For humans, the front-end must provide high visual credibility (neon-accented dark mode, clear metrics, and responsive glassmorphism) and smooth wallet UX.

### A. The LP Syndicate Vault Portal
*   **Purpose**: Where capital providers deposit assets to back transaction risks and earn premium yields.
*   **Key Features**:
    *   **TVL & Yield Analytics**: Real-time counter showing Total Value Locked (staked BTC, WBTC, stablecoins) and average historical APY generated from underwriting fees.
    *   **Liquidation Bounty Tracker**: A dedicated counter showing the active **Liquidation Bounty yield** (the dynamic 5% penalty bonus collected from defaulting merchants' locked stake) earned by the LP on top of standard underwriting premiums, illustrating the direct incentive for backing risk.
    *   **Dynamic Deposit/Withdrawal Module**: Smooth slider inputs displaying current lock-up cooldowns and warning banners explaining **FR-1.4 (Collateral Reservation)**—explaining how much capital is currently "locked" to back active policies.
    *   **Pro-rata Yield Calculator**: Simulates projected earnings based on deposit volume and syndicate risk levels.

### B. The Merchant Stake Vault & Developer Console
*   **Purpose**: Where merchants manage their checkout integration and maintain their collateral vaults to qualify for instant settlements.
*   **Key Features**:
    *   **Merchant Trust Dashboard**: Displays the merchant's current **quantifiable Trust Score** (e.g. `92/100`), showing real-time feedback on dispute rates, wallet cluster status, and clearing history.
    *   **Stake Vault Collateral Locker**: Interactive module displaying the merchant's locked **USD Stablecoin (USDT/USDC)** collateral in their Stake Vault vs. the required safety threshold to maintain $T+0$ instant payout privileges.
    *   **Reimbursement & Grace Alerts**: Displays high-priority visual banners when customer returns are processed, showing an active **48-hour countdown timer** displaying exactly how much time is left before programmatic collateral liquidation triggers to reimburse the Syndicate Vault.
    *   **x402 Dynamic Premium Tracker**: Displays their active risk premium rate (e.g., standard 0.5% vs. spiked 5% default rate).
    *   **No-Code Integration Wizard**: A copy-paste generator where developers select their target blockchain (e.g. Base), input their smart contract address, and instantly generate a protected button:
        ```jsx
        <NovaeRogShield 
          contractAddress="0x..." 
          coveredAmount={5000} 
          currency="USDT" 
        />
        ```

### C. The Coverage & Claims Explorer
*   **Purpose**: A public blockchain-explorer-like interface to ensure complete transparency of the mutual pool.
*   **Key Features**:
    *   **Live Coverage Stream**: A feed of active insured transactions showing the covered transaction hash, the premium paid, and time remaining until coverage expiration.
    *   **Direct On-Chain Signature Proofs**: Displays the verified cryptographic signatures generated by the agents (`ROSA_Sig`, `XCPA_Sig`, `CVAA_Sig`) that are validated directly on-chain by the smart contracts, proving the zero-trust security of the database layer.
    *   **Audit Multi-Sig Panel**: A dedicated interface for human signers to review claims >= $50M. It displays the AI agent's raw verification logs, explorer verification proof, and a single "Approve Claim" button that triggers MetaMask/Rabby to sign the Gnosis Safe transaction.

### D. The Real-Time "Air-Gap Pipeline" Visualizer (GUI)
*   **Purpose**: An animated, graphical dashboard used to visually demonstrate the secure, multi-agent air-gapped clearing pipeline to hackathon judges.
*   **Key Features**:
    *   **The Forest Council Nodes**: Beautiful, glowing graphical cards representing the specialized agents: **ROSA 🦉 (Risk Oracle)**, **XCPA 🐿️ (micropayments)**, **CVAA 🦝 (Arbitration)**, and **VRSA 🐻 (Vault Settlement)**.
    *   **Data Density Overlay**: All graphical animations act as an enhancement, never as a replacement for raw data. The visual nodes explicitly display real-time **quantifiable data** (numeric trust scores, active wallet cluster sizes, raw premium values in Wei/USDT, transaction block heights, API latencies, and truncated cryptographic hashes) to guarantee technical transparency and prove engineering authenticity.

## 🤖 2. The Machine-to-Machine Interface Layer (Agentic Discovery)

AI agents do not browse standard websites. To allow OpenClaw, Eliza, or LangChain agents to discover and interact with Novae Rog autonomously, the protocol must implement a semantic discovery layer:

### A. Model Context Protocol (MCP) Server Integration
*   **Purpose**: A standard open-source protocol that allows LLM models (like Claude or Gemini) to securely discover and execute tools.
*   **How it works**: Expose an MCP server endpoint defining core tools:
    *   `get_underwriting_quote(targetContract, coveredAmount, token)`: Returns the premium quote.
    *   `purchase_cover(quoteId, paymentTxHash)`: Registers active protection.
    *   `file_transaction_claim(txHash)`: Submits programmatic claim evidence.
*   **Benefit**: An autonomous AI trading agent can read these schema definitions and immediately execute transaction insurance programmatically in its workflow without needing any human-designed UI.

### B. Semantic Web Registry (`/agentic-services.json`)
*   **Purpose**: An agentic equivalent of `robots.txt` hosted at the root domain.
*   **How it works**: The website hosts a JSON schema at `/agentic-services.json` containing:
    *   Descriptions of the protocol's purpose in natural language (for LLM indexing).
    *   Smart contract ABI locations and active deployment addresses across GOAT, Base, and Solana networks.
    *   Standard REST API OpenAPI/Swagger schemas.
*   **Benefit**: Web-crawler agents can catalog Novae Rog under "Agent Risk Management" registries automatically.

### C. EIP-712 Structured Signing
*   **Purpose**: Standardized on-chain message structures.
*   **How it works**: Quotes and active coverage tickets are formatted as EIP-712 structured data. 
*   **Benefit**: Agents can cryptographically sign agreement payloads programmatically using their hot wallets, ensuring highly secure, tamper-proof, and gas-efficient on-chain execution.
